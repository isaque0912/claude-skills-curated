# Contributing to Claude Skills Curated

Thanks for your interest in contributing! This is a curated list, so we're selective about what gets added.

## Suggesting a Resource

Open an issue with:

1. **Link** - URL to the GitHub repo or resource
2. **Description** - What does it do? (1-2 sentences)
3. **Why it's great** - What makes it stand out?
4. **Category** - Where should it go in the list?

## Quality Criteria

We evaluate submissions based on:

| Criteria | What we look for |
|----------|------------------|
| **Usefulness** | Solves real problems for Claude Code users |
| **Quality** | Well-documented, follows best practices |
| **Maintenance** | Recently updated, issues addressed |
| **Popularity** | Stars, forks, community adoption |
| **Uniqueness** | Offers something others don't |

## What We Don't Accept

- Abandoned projects (no updates in 6+ months)
- Poorly documented repos
- Duplicates of existing entries
- Self-promotion without substance
- Resources unrelated to Claude Code/Skills

## Rating Guidelines

When suggesting a rating:

- **⭐⭐⭐ Essential** - Every Claude Code user should know about this
- **⭐⭐ Excellent** - High quality, recommended for most users
- **⭐ Good** - Useful for specific use cases or niches

## Pull Requests

You can also submit a PR directly:

1. Fork the repo
2. Add your entry in the appropriate category
3. Follow the existing table format
4. Include a brief, accurate description
5. Submit PR with explanation of why it belongs

## Questions?

Open an issue and we'll help out.
